## 運用ルール

- 結果確定は `races.status=final` または `race_result_status.status=final` を使用する。
- F、返還、不成立は `trifecta_evaluable=false` として通常の3連単評価から除外し、結果待ちには残さない。
- Web、collector、予測、shadow停止時だけ運用懸案を再オープンする。
- 新規番号付き、`vN`、`safeN` Pythonモジュールを追加しない。
- 実装完了、評価完了、性能ゲート通過を別の状態として扱う。
- 数値はPostgreSQL、成果物JSON、Supervisorプロセスから動的に取得する。
