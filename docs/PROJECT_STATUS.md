## 運用監視

- 結果確定は着順行数だけでなく、`races.status=final` または `race_result_status.status=final` も使用する。
- F、返還、不成立は `trifecta_evaluable=false` として通常の3連単評価から除外し、結果待ち・監視中には残さない。
- 実odds shadowは30分周期で対象件数を再判定する。
- Web、collector、predictor、shadowの停止時だけ復旧懸案を再オープンする。
- 新規番号付き、`vN`、`safeN` Pythonモジュールを追加しない。

- 実装完了、評価完了、性能ゲート通過を別の状態として扱う。
- 評価が終わっても性能ゲート未達なら、本番昇格やM6完了にはしない。
- 数値はDB、成果物JSON、`data/remote_eval_status.json`、常駐プロセスから動的に取得する。
- 静的PIDや実行中だった過去状態を現在状態として記録しない。
