## 結論

難読化コード、動的評価、資格情報の第三者宛送信、暗号資産採掘、永続化処理などの明確な不正コードは検出しなかった。外部通信先は、アプリ本体ではTeleboatの `https://mb.brtb.jp/`、ビルド時はRubyGems、Google Chrome配布元、ChromeDriver配布元である。

ただし、原版は実投票を直接確定するサービスであり、認証、重複防止、上限管理、結果確認、ブラウザの後始末に重大な運用リスクがある。そのまま稼働させるべき状態ではない。

## Python版

実装場所は `src/teleboat_agent/`。

- `models.py`: 場コード、レース番号、三連単組番、100円単位数量、合計投入額を検証
- `config.py`: 秘密情報を環境変数から取得し、live有効時の必須値を起動時検証
- `service.py`: dry-run、ライブ三重ゲート、冪等性、バッチ分割
- `browser.py`: Seleniumを任意依存にし、例外時も必ず終了
- `api.py`: 原版相当のPOST経路をBearer認証付きWSGI APIとして提供

原版相当の経路は `POST /api/internal/v1/tickets/votes`。dry-run応答では、実際に送るコード、点数、投入額、バッチを確認できる。

ログイン専用probeは、ローカルChrome headlessで移植元と同じ公式 `mb.brtb.jp` を使用し、ログイン成功と公式ログアウト完了URL到達を確認した。資格情報はダッシュボードのlocalhost/ローカルHTTPS限定画面から0600ファイルへ保存し、Web/APIには値を返さない。さらに若松8R・3連単2-4-1・数量1をベットリストへ追加し、投票直前画面で1ベット・購入金額100円・投票未完了を確認した。最終「投票する」ボタンは押さず、投票成立0件。最終送信後の成立判定、返却画面、重複防止の実サイト検証まではliveを有効化しないこと。
